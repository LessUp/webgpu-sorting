example.ts 
import { GPUContext, BitonicSorter } from 'webgpu-sorting';

// Initialize WebGPU context
const gpu = new GPUContext();
await gpu.initialize();

// Create sorter
const sorter = new BitonicSorter(gpu);

// Sort data on GPU
const data = new Uint32Array([5, 2, 8, 1, 9]);
const { sortedData } = await sorter.sort(data);

Why this repo is useful

WebGPU sorting reference

Two algorithm paths

Bitonic Sort is the clean sorting-network baseline, while Radix Sort is the better fit for large integer-heavy workloads.

What is implemented

Bitonic Sort

O(n log²n)

A comparison-based parallel sorting network that makes the GPU execution pattern easy to inspect and reason about.

  • Good baseline for browser-side GPU sorting experiments
  • Handles padding to power-of-two sizes
  • Best fit: general-purpose reference implementation

Radix Sort

O(n × k)

A non-comparison integer sorter built around histogram and prefix-sum style passes for large fixed-width integer arrays.

  • Specialized for Uint32Array workloads
  • Includes GPU/CPU cooperation for simplified prefix sums
  • Best fit: large integer-heavy datasets
